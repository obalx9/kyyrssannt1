## Вариант А: Если у вас ДВА поля

```
┌─────────────────────────────────────────────┐
│  Команды сборки и запуска                   │
├─────────────────────────────────────────────┤
│                                             │
│  Команда сборки (Build Command):            │
│  ┌───────────────────────────────────────┐  │
│  │ npm install && npm run build          │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  Команда запуска (Start Command):           │
│  ┌───────────────────────────────────────┐  │
│  │ npm start                             │  │
│  └───────────────────────────────────────┘  │
│                                             │
└─────────────────────────────────────────────┘
```

### ✅ Что происходит:
1. **При деплое:** `npm install && npm run build`
   - Устанавливает зависимости
   - Собирает React проект с помощью Vite
   - Создаёт папку `build/` со статикой

2. **Постоянно работает:** `npm start`
   - Запускает Node.js сервер
   - Отдаёт статику из `build/`
   - Обрабатывает API запросы

## Переменные окружения

После основных настроек вы увидите раздел с переменными:

```
┌─────────────────────────────────────────────┐
│  Переменные окружения                       │
├─────────────────────────────────────────────┤
│                                             │
│  [+ Добавить переменную]                    │
│                                             │
│  Ключ                  Значение             │
│  ────────────────────  ──────────────────   │
│  PORT                  3000                 │
│  NODE_ENV              production           │
│  DATABASE_URL          postgresql://...     │
│  JWT_SECRET            a7f2c8e9b1d4f6a3... │
│  ALLOWED_ORIGINS       https://yoursite.ru  │
│                                             │
└─────────────────────────────────────────────┘
```

### 📝 Добавьте эти переменные:

| Ключ | Значение | Описание |
|------|----------|----------|
| `PORT` | `3000` | Порт сервера |
| `NODE_ENV` | `production` | Режим работы |
| `DATABASE_URL` | `postgresql://user:pass@host:5432/db` | Строка подключения к БД |
| `JWT_SECRET` | `[сгенерированная строка 64+ символа]` | Секретный ключ для JWT |
| `ALLOWED_ORIGINS` | `https://yoursite.ru,https://www.yoursite.ru` | Разрешённые домены |

## 🔧 Типичные проблемы

### Проблема 1: "vite: not found"

**Симптом:**
```
Error: vite: command not found
```

**Причина:** Vite в devDependencies, а на проде устанавливаются только dependencies

**Решение:** Мы уже это исправили! Vite теперь в `dependencies`

---

### Проблема 2: "Missing Supabase environment variables"

**Симптом:**
```
Error: Missing required Supabase environment variables
```

**Причина:** Старая сборка требует Supabase

**Решение:**
```bash
npm run build
git add build/
git commit -m "Update build"
git push
```

---

### Проблема 3: "CORS error"

**Симптом:**
```
Access to fetch blocked by CORS policy
```

**Причина:** Неправильно настроен ALLOWED_ORIGINS

**Решение:** Добавьте в переменные окружения:
```
ALLOWED_ORIGINS=https://your-domain.ru,https://www.your-domain.ru
```

- [ ] `vite` находится в `dependencies` (не в devDependencies)
- [ ] Проект собирается локально: `npm run build`
- [ ] Папка `build/` создаётся успешно
- [ ] Все изменения закоммичены в git

- [ ] В Timeweb созданы все переменные окружения
