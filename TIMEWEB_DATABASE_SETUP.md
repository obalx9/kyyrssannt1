# Настройка базы данных PostgreSQL на Таймвеб

## Быстрая диагностика через браузер

После запуска сервера откройте в браузере:
```
http://ваш-домен.ru/api/diagnostics
```

Эта страница позволит вам:
- Проверить подключение к базе данных
- Посмотреть список созданных таблиц
- Проверить переменные окружения
- Выполнить полную диагностику системы

## Шаг 1: Создание базы данных PostgreSQL

1. Войдите в панель управления Таймвеб
2. Перейдите в раздел "Базы данных"
3. Нажмите "Создать базу данных"
4. Выберите тип: **PostgreSQL**
5. Укажите имя базы данных (например: `kursat_db`)
6. Создайте пользователя и пароль
7. Сохраните данные подключения

## Шаг 2: Настройка переменных окружения

Создайте или отредактируйте файл `.env` на сервере:

```env
DATABASE_URL=postgresql://username:password@host:port/database_name
JWT_SECRET=ваш_секретный_ключ_минимум_32_символа
NODE_ENV=production
PORT=3000
ALLOWED_ORIGINS=https://ваш-домен.ru
```

### Пример строки подключения:
```
DATABASE_URL=postgresql://user123:mypassword@pg-12345.timeweb.cloud:5432/kursat_db
```

Компоненты строки подключения:
- `username` - имя пользователя БД
- `password` - пароль пользователя
- `host` - адрес сервера PostgreSQL (обычно заканчивается на .timeweb.cloud)
- `port` - порт (обычно 5432)
- `database_name` - имя базы данных

## Шаг 3: Применение миграций через панель Таймвеб

### Важно: Применяйте миграции строго по порядку!

#### 3.1. Откройте SQL редактор

1. В панели Таймвеб перейдите в "Базы данных"
2. Выберите вашу базу PostgreSQL
3. Найдите раздел "SQL редактор" или "Выполнить SQL"

#### 3.2. Применение миграции 1: Система аутентификации

Откройте файл `timeweb-migrations/01_create_auth_system.sql`

Скопируйте весь содержимое файла и вставьте в SQL редактор.

Нажмите "Выполнить" или "Execute".

**Что создается:**
- Таблица `users` - пользователи системы
- Таблица `user_roles` - роли пользователей
- Индексы для быстрого поиска

#### 3.3. Применение миграции 2: Схема платформы

Откройте файл `timeweb-migrations/02_create_platform_schema.sql`

Скопируйте и выполните в SQL редакторе.

**Что создается:**
- Таблица `sellers` - продавцы курсов
- Таблица `students` - студенты
- Таблица `courses` - курсы
- Таблица `course_posts` - посты в курсах
- Таблица `course_post_media` - медиафайлы
- Таблица `course_enrollments` - записи студентов на курсы
- Таблица `student_pinned_posts` - закрепленные посты
- Таблица `telegram_bots` - Telegram боты
- Таблица `telegram_media_group_buffer` - буфер для медиа-групп

#### 3.4. Применение миграции 3: Политики безопасности

Откройте файл `timeweb-migrations/03_setup_rls_policies.sql`

Скопируйте и выполните в SQL редакторе.

**Что создается:**
- Row Level Security (RLS) политики для защиты данных
- Права доступа для пользователей

#### 3.5. Применение миграции 4: Дополнительные функции

Откройте файл `timeweb-migrations/04_add_additional_features.sql`

Скопируйте и выполните в SQL редакторе.

**Что создается:**
- Дополнительные функции для работы с курсами
- Триггеры для автоматических действий
- Дополнительные индексы

## Распространенные ошибки и решения

### Ошибка: "relation does not exist"

**Причина:** Миграции не были применены или применены не полностью.

**Решение:**
1. Откройте `/api/db-tables` в браузере
2. Проверьте количество таблиц
3. Если таблиц меньше 11, примените недостающие миграции

### Ошибка: "connection refused" или "timeout"

**Причина:** Неверная строка подключения DATABASE_URL.

**Решение:**
1. Проверьте `.env` файл
2. Убедитесь, что адрес сервера БД правильный
3. Проверьте, что порт указан верно (обычно 5432)
4. Убедитесь, что нет пробелов в строке подключения

### Ошибка: "password authentication failed"

**Причина:** Неверный логин или пароль.

**Решение:**
1. Проверьте учетные данные в панели Таймвеб
2. Обновите DATABASE_URL с правильным паролем
3. Перезапустите сервер

### Ошибка: "database does not exist"

**Причина:** База данных не создана или указано неверное имя.

**Решение:**
1. Создайте базу данных в панели Таймвеб
2. Проверьте имя базы в DATABASE_URL

## Дополнительные команды для проверки через SQL редактор

### Проверка всех таблиц:
```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

### Проверка количества записей:
```sql
SELECT
  'users' as table_name, COUNT(*) as count FROM users
UNION ALL SELECT 'courses', COUNT(*) FROM courses
UNION ALL SELECT 'sellers', COUNT(*) FROM sellers
UNION ALL SELECT 'students', COUNT(*) FROM students
UNION ALL SELECT 'telegram_bots', COUNT(*) FROM telegram_bots;
```

### Проверка структуры таблицы:
```sql
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'users'
ORDER BY ordinal_position;
```
