## Как запустить локально

### 1. Подготовка

```bash
# Установить зависимости Frontend
npm install

# Установить зависимости API
cd backend-api && npm install && cd ..
```

### 2. Запустить в разных терминалах

**Terminal 1 - API сервер:**
```bash
cd backend-api
npm start
```
Откроется на `http://localhost:3000`

**Terminal 2 - Frontend:**
```bash
npm run dev
```
Откроется на `http://localhost:5173`

### 3. Тестирование

```bash
# Health check API
curl http://localhost:3000/health

# Проверка БД
curl http://localhost:3000/api/db-check
```

## Как развернуть на Timeweb

### 1. Frontend (Статический сайт)

```bash
# Собрать
npm run build

# Результат: папка build/
# Загрузить в Timeweb Apps → Static Website
```

**На Timeweb Apps:**
1. Apps → Создать → Static Website
2. Выбрать папку `build/`
3. Привязать домен: `keykurs.ru`

### 2. Backend API (Node.js)

```bash
# Загрузить папку backend-api/ на Timeweb Apps
```

**На Timeweb Apps:**
1. Apps → Создать → Backend → Node.js
2. Загрузить папку `backend-api/`
3. Добавить переменные окружения в Settings:
   - `DATABASE_URL`
   - `JWT_SECRET`
   - `PORT` (обычно 8000 на Timeweb)
   - `ALLOWED_ORIGINS`
4. Привязать домен: `api.keykurs.ru`

### 3. База данных (PostgreSQL)

Уже создана на отдельном сервере Timeweb.

## Переменные окружения

### Общий файл `.env` (в корне проекта)

```env
# Frontend
VITE_SUPABASE_URL=https://...
VITE_SUPABASE_ANON_KEY=...

# API (используется в backend-api/)
DATABASE_URL=postgresql://user:pass@host:5432/db?sslmode=require
JWT_SECRET=your_secret_key
PORT=3000
NODE_ENV=production
ALLOWED_ORIGINS=http://localhost:5173,https://keykurs.ru
```

### Для Timeweb Apps

**Frontend:**
- Не требует переменных (статический сайт)

**Backend:**
- Добавить в Settings → Environment Variables
- `DATABASE_URL`
- `JWT_SECRET`
- `ALLOWED_ORIGINS`
- `PORT=8000` (для Timeweb)

## Важные моменты

### ✅ Правильно

- API получает запросы от Frontend по HTTP
- API подключается к БД через connectionString в `.env`
- Креды БД хранятся только на сервере API
- Frontend НЕ имеет доступа к БД
- Каждый сервис имеет собственный каталог

### ❌ Неправильно

- Подключаться к БД напрямую из браузера
- Хранить DATABASE_URL в коде React
- Совмещать Frontend и API в одной папке

## Переход со старой структуры

**Старая структура:**
```
project/
├── src/           # Frontend
├── api/           # API (в одной папке!)
└── ...
```

**Новая структура:**
```
project/
├── src/           # Frontend
├── backend-api/   # API (отдельный каталог!)
└── ...
```

**Что изменилось:**
- Папка `api/` → `backend-api/`
- Отдельный `npm install` для API
- Отдельный процесс запуска
- Проще развертывать на разных серверах

## Вопросы и ответы

**Q: Как изменить порт API?**
A: Меняйте переменную `PORT` в `.env`

**Q: Где хранить переменные окружения?**
A: В файле `.env` (локально) и в Settings Apps (на Timeweb)

**Q: Может ли Frontend работать без API?**
A: Да, но без доступа к БД. Используйте только локальное хранилище.

**Q: Как обновить API на Timeweb?**
A: Загрузить новую версию папки `backend-api/` через Apps

**Q: Почему отдельные сервисы лучше?**
A: Масштабируемость, безопасность, независимые развертывания
