📋 Этапы настройки

1 Создание базы данных PostgreSQL

В панели Таймвеб:

  • Перейдите в раздел "Базы данных"
  • Нажмите "Создать базу данных"
  • Выберите тип: PostgreSQL
  • Укажите имя и сохраните данные подключения

2 Создание файла .env

Создайте файл .env в корне проекта:

DATABASE_URL=postgresql://user:password@host:5432/database JWT_SECRET=ваш_секрет_минимум_32_символа NODE_ENV=production PORT=3000 ALLOWED_ORIGINS=https://ваш-домен.ru

3 Применение SQL миграций

В SQL редакторе Таймвеб выполните по порядку:

  1. timeweb-migrations/01_create_auth_system.sql
  2. timeweb-migrations/02_create_platform_schema.sql
  3. timeweb-migrations/03_setup_rls_policies.sql
  4. timeweb-migrations/04_add_additional_features.sql

4 Настройка Telegram бота

Создайте бота через @BotFather и добавьте в БД:

INSERT INTO telegram_bots (bot_username, bot_token, is_active) VALUES ('YourBot', 'token_from_botfather', true);

❌ Частые ошибки и решения

Ошибка: connection refused

Проверьте DATABASE_URL в файле .env. Убедитесь, что адрес сервера, порт и учетные данные указаны правильно.

Ошибка: relation does not exist

Миграции не применены. Выполните все SQL файлы из папки timeweb-migrations по порядку.

Ошибка: password authentication failed

Неверный пароль в DATABASE_URL. Проверьте пароль в панели Таймвеб и обновите .env файл.
