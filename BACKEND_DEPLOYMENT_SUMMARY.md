# 🎯 Итоговая инструкция: Backend API на Timeweb Cloud Apps

## ✅ Всё готово к деплою!

Подготовлена полная документация для развертывания backend-api как отдельного приложения на Timeweb Cloud Apps.

---

## 📦 Что деплоится

**Папка:** `/backend-api`
- Node.js приложение с Express фреймворком
- Все API эндпоинты (авторизация, курсы, посты, estudiantes и т.д.)
- Интеграция с PostgreSQL БД на Timeweb
- Telegram webhook обработка
- Загрузка медиафайлов

**Результат:** Отдельное приложение на `https://your-backend.timeweb.app`

## ⏱️ Быстрый старт (5 минут)

### Шаг 1: Скопируйте значения
Откройте `BACKEND_DEPLOY_COPY_PASTE.txt` и скопируйте:
- Команду сборки
- Команду запуска
- Путь проверки

### Шаг 2: Получите DATABASE_URL
1. Войдите в Timeweb → Облачные БД → PostgreSQL
2. Нажмите на вашу БД
3. Скопируйте "Connection String"

### Шаг 3: Сгенерируйте JWT_SECRET
```bash
openssl rand -hex 32
# Скопируйте результат
```

### Шаг 4: В панели Timeweb
1. Облачные приложения → Создать приложение
2. Выбрать Node.js 24
3. Выбрать Express
4. Подключить GitHub (obalx9/kyyrssannt, ветка main)
5. Заполнить команды и переменные
6. Нажать "Развернуть"

### Шаг 5: Проверить
```bash
curl https://your-api.timeweb.app/health
# Должно вернуть: {"status":"ok","timestamp":"..."}
```

**Готово!** ✅ Backend работает!

## 🔗 Как это работает с фронтенд-приложением

### На фронтенде (.env или переменные Timeweb)
```
VITE_API_URL=https://your-backend.timeweb.app
```

### На фронтенде (код React)
```javascript
const response = await fetch(
  `${import.meta.env.VITE_API_URL}/api/courses`,
  {
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    }
  }
);
```

### На бэкенде (переменная окружения)
```
ALLOWED_ORIGINS=https://your-frontend.timeweb.app,http://localhost:5173
```

**Результат:** Фронтенд и бэкенд общаются через API! ✅

## 🐛 Типичные проблемы

| Проблема | Решение |
|----------|---------|
| 502 Bad Gateway | Подождите 2-3 минуты (деплой в процессе) или проверьте логи |
| Cannot connect to database | Проверьте DATABASE_URL (скопируйте заново из БД) |
| CORS error | Проверьте ALLOWED_ORIGINS (должен быть ваш домен фронта) |
| Приложение перезагружается | Проверьте логи на ошибки, исправьте, пушьте в main |
| Переменная не находится | Перезагрузите приложение в панели или сделайте новый деплой |

**Больше проблем?** → `BACKEND_FAQ.md`
