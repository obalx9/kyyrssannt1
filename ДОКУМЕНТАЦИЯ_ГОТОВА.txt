📋 ГОТОВЫЕ ЗНАЧЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

Окружение:      Node.js 24
Фреймворк:      Express
Сборка:         npm install
Запуск:         pm2 start --no-daemon index.js
Проверка:       /health
Ветка:          main

Переменные (5):
1. DATABASE_URL      ← postgresql://... (из Timeweb БД)
2. JWT_SECRET        ← Сгенерировать (openssl rand -hex 32)
3. NODE_ENV          ← production
4. PORT              ← 3000
5. ALLOWED_ORIGINS   ← https://your-domain.com (фронтенд)

💡 НЕСКОЛЬКО СОВЕТОВ
═══════════════════════════════════════════════════════════════════════════════

1. НЕ коммитьте реальные переменные в .env файлы
   └─ Храните только в панели Timeweb (они зашифрованы)

2. DATABASE_URL содержит пароль - это нормально
   └─ Переменные окружения безопасны в Timeweb

3. JWT_SECRET должен быть криптостойким
   └─ Используйте openssl rand -hex 32 (не простые пароли!)

4. ALLOWED_ORIGINS должен быть точный домен фронтенда
   └─ Включая https:// и без слешей в конце

5. При обновлении кода в main ветке Timeweb переделает автоматически
   └─ Не нужно ничего вручную!
