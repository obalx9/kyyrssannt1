# Проверка подключения к базе данных

## Самый быстрый способ

Откройте в браузере (замените на ваш домен):
```
https://ваш-домен.ru/api/diagnostics
```

Нажмите кнопку **"Запустить полную проверку"**.

## Альтернативные способы проверки

### Способ 1: Прямой запрос к API

Откройте в браузере:
```
https://ваш-домен.ru/api/db-check
```

**Успешный ответ:**
```json
{
  "status": "connected",
  "connection": "ok",
  "database": "имя_вашей_бд",
  "serverTime": "2024-...",
  "postgresVersion": "PostgreSQL 14...",
  "tablesCount": 11,
  "message": "База данных подключена успешно"
}
```

**Ошибка подключения:**
```json
{
  "status": "error",
  "connection": "failed",
  "error": "connect ECONNREFUSED",
  "message": "Ошибка подключения к базе данных",
  "hint": "Проверьте DATABASE_URL в файле .env"
}
```

### Способ 2: Проверка таблиц

```
https://ваш-домен.ru/api/db-tables
```

**Успешный ответ:**
```json
{
  "status": "success",

    {"name": "courses", "columns": 12, "rows": 0},
```

### Способ 3: Проверка переменных

**Успешный ответ:**

## Что делать если проверка не прошла?

### ❌ Ошибка: База данных не подключается

**Причины:**
- DATABASE_URL не установлен в .env
- Неверные учетные данные
- База данных не запущена
- Неверный адрес сервера

**Решение:**
1. Проверьте файл `.env`
2. Убедитесь в формате: `postgresql://user:password@host:5432/database`
3. Проверьте данные в панели Таймвеб (раздел "Базы данных")
4. Перезапустите приложение

### ⚠️ Ошибка: Таблиц не найдено (0 таблиц)

**Причина:** SQL миграции не были применены к базе данных.

**Решение:**
1. Откройте панель Таймвеб
2. Перейдите в "Базы данных" → ваша БД → SQL редактор
3. Примените миграции по порядку:
   - `timeweb-migrations/01_create_auth_system.sql`
   - `timeweb-migrations/02_create_platform_schema.sql`
   - `timeweb-migrations/03_setup_rls_policies.sql`
   - `timeweb-migrations/04_add_additional_features.sql`
4. Проверьте снова через `/api/db-check`

### ⚠️ Ошибка: JWT_SECRET не установлен

**Решение:**
1. Откройте файл `.env`
2. Добавьте строку:
   ```
   JWT_SECRET=ваш_случайный_секрет_минимум_32_символа
   ```
3. Сохраните и перезапустите приложение

## Проверка через SQL редактор Таймвеб

Если API недоступен, проверьте напрямую через SQL редактор:

### Проверка таблиц:
```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

**Должно быть 11 таблиц:**
- course_enrollments
- course_post_media
- course_posts
- courses
- sellers
- student_pinned_posts
- students
- telegram_bots
- telegram_media_group_buffer
- user_roles
- users

### Проверка количества записей:
```sql
SELECT
  'users' as table_name, COUNT(*) FROM users
UNION ALL SELECT 'courses', COUNT(*) FROM courses
UNION ALL SELECT 'sellers', COUNT(*) FROM sellers
UNION ALL SELECT 'students', COUNT(*) FROM students
UNION ALL SELECT 'telegram_bots', COUNT(*) FROM telegram_bots;
```

### Проверка версии PostgreSQL:
```sql
SELECT version();
```

## Быстрая справка по эндпоинтам

| Эндпоинт | Что проверяет | Требует авторизацию |
|----------|---------------|---------------------|

| `/api/diagnostics` | Интерактивная страница с кнопками проверки | Нет |
| `/api/telegram-bot` | Настроенный Telegram бот | Нет |
| `/health` | Работоспособность сервера | Нет |
