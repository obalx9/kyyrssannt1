# 🚀 Быстрый старт - Проверка связи с БД

## Для чего это нужно?

Ваш сайт работает на российском хостинге Таймвеб и использует PostgreSQL базу данных. Эта инструкция поможет вам проверить, что сайт правильно подключен к базе данных.

## Шаг 1: Запустите сервер

На хостинге Таймвеб запустите ваше приложение. Обычно это делается командой:
```bash
npm start
```

Или через панель управления приложениями Таймвеб.

## Шаг 2: Откройте страницу диагностики

В браузере откройте:
```
http://ваш-домен.ru/api/diagnostics
```

Замените `ваш-домен.ru` на ваш реальный домен.

## Шаг 3: Запустите проверки

На странице диагностики вы увидите несколько кнопок:

### 🔌 Проверить подключение
Нажмите эту кнопку первой. Она покажет:
- Подключается ли сайт к PostgreSQL
- Версию PostgreSQL
- Имя базы данных
- Количество таблиц

**Ожидаемый результат:** ✅ УСПЕШНО ПОДКЛЮЧЕНО

### 📋 Показать таблицы
Показывает список всех таблиц в базе данных и количество записей в них.

**Ожидаемый результат:** Список из 11+ таблиц (users, courses, sellers и т.д.)

### ⚙️ Проверить конфигурацию
Проверяет, что все необходимые переменные окружения установлены.

**Ожидаемый результат:** Все необходимые переменные окружения установлены

### 🚀 Запустить полную проверку
Выполняет все проверки сразу и выдает итоговый отчет.

## Что делать, если проверки не проходят?

### ❌ Ошибка подключения к БД

**Проблема:** DATABASE_URL не настроен или указан неверно.

**Решение:**
1. Откройте файл `.env` на сервере
2. Проверьте строку DATABASE_URL
3. Формат должен быть:
   ```
   DATABASE_URL=postgresql://user:password@host:5432/database_name
   ```
4. Получите правильные данные из панели Таймвеб (раздел "Базы данных")

### ⚠️ Таблиц не найдено

**Проблема:** SQL миграции не были применены к базе данных.

**Решение:**
1. Откройте панель Таймвеб
2. Перейдите в "Базы данных" → выберите вашу БД
3. Откройте SQL редактор
4. Примените файлы миграций по порядку:
   - `timeweb-migrations/01_create_auth_system.sql`
   - `timeweb-migrations/02_create_platform_schema.sql`
   - `timeweb-migrations/03_setup_rls_policies.sql`
   - `timeweb-migrations/04_add_additional_features.sql`

Подробная инструкция: см. файл `TIMEWEB_DATABASE_SETUP.md`

### ⚠️ JWT_SECRET не установлен

**Проблема:** Отсутствует секретный ключ для JWT токенов.

**Решение:**
1. Откройте файл `.env` на сервере
2. Добавьте строку:
   ```
   JWT_SECRET=ваш_случайный_секретный_ключ_минимум_32_символа
   ```
3. Сгенерируйте случайную строку длиной 32+ символа

## Альтернативные способы проверки

Если страница диагностики не открывается, можно проверить через API напрямую:

### Проверка подключения к БД:
```
GET http://ваш-домен.ru/api/db-check
```

Откройте эту ссылку в браузере. Ожидаемый ответ:
```json
{
  "status": "connected",
  "database": "имя_вашей_бд",
  "tablesCount": 11
}
```

### Проверка таблиц:
```
GET http://ваш-домен.ru/api/db-tables
```

### Проверка переменных:
```
GET http://ваш-домен.ru/api/env-check
```

## Важные файлы и документация

- `TIMEWEB_DATABASE_SETUP.md` - Полная инструкция по настройке БД
- `timeweb-migrations/` - Папка с SQL миграциями
- `.env` - Файл с переменными окружения (создайте если нет)
- `/api/diagnostics` - Страница диагностики (открывается в браузере)

## Поддержка

При возникновении проблем:
1. Запустите полную диагностику на странице `/api/diagnostics`
2. Скопируйте результаты
3. Проверьте логи сервера
4. Убедитесь, что все миграции применены
