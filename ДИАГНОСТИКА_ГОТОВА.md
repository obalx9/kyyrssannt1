# ✅ Система диагностики установлена

## Что было добавлено?

В ваш проект добавлена полная система диагностики для проверки подключения к PostgreSQL базе данных на хостинге Таймвеб **через браузер**, без необходимости использовать терминал.

---

## 🎯 Главное: Страница диагностики

После запуска сервера откройте в браузере:

```
https://ваш-домен.ru/api/diagnostics
```

На этой странице доступны:

### 1. 🔌 Проверка подключения к БД
Показывает:
- Статус подключения (успешно/ошибка)
- Версию PostgreSQL
- Имя базы данных
- Время сервера
- Количество таблиц

### 2. 📋 Структура таблиц
Показывает:
- Список всех таблиц в базе
- Количество колонок в каждой таблице
- Количество записей в каждой таблице

### 3. ⚙️ Переменные окружения
Проверяет:
- Наличие DATABASE_URL
- Наличие JWT_SECRET
- NODE_ENV
- PORT
- ALLOWED_ORIGINS

### 4. 🚀 Полная диагностика
Запускает все проверки сразу и выдает итоговый отчет с рекомендациями.

## 📡 API Эндпоинты для проверки

Добавлены новые эндпоинты, которые можно вызывать напрямую:

### GET /api/db-check
Проверяет подключение к базе данных.

**Пример запроса:**
```
https://ваш-домен.ru/api/db-check
```

**Успешный ответ:**
```json
{
  "status": "connected",
  "connection": "ok",
  "database": "keykurs",
  "serverTime": "2024-02-18T10:30:00.000Z",
  "postgresVersion": "PostgreSQL 14.5 on x86_64-pc-linux-gnu...",
  "tablesCount": 11,
  "message": "База данных подключена успешно"
}
```

**Ответ при ошибке:**
```json
{
  "status": "error",
  "connection": "failed",
  "error": "connect ECONNREFUSED 127.0.0.1:5432",
  "code": "ECONNREFUSED",
  "message": "Ошибка подключения к базе данных",
  "hint": "Проверьте DATABASE_URL в файле .env"
}
```

### GET /api/db-tables
Возвращает список всех таблиц с метаданными.

**Пример запроса:**
```
https://ваш-домен.ru/api/db-tables
```

**Ответ:**
```json
{
  "status": "success",
  "totalTables": 11,
  "tables": [
    {
      "name": "users",
      "columns": 7,
      "rows": 0
    },
    {
      "name": "courses",
      "columns": 12,
      "rows": 5
    },
    ...
  ],
  "message": "Структура базы данных получена"
}
```

### GET /api/env-check
Проверяет переменные окружения (без показа значений для безопасности).

**Пример запроса:**
```
https://ваш-домен.ru/api/env-check
```

**Ответ:**
```json
{
  "status": "ok",
  "environment": {
    "DATABASE_URL": true,
    "DATABASE_URL_preview": "postgresql://user:...",
    "JWT_SECRET": true,
    "NODE_ENV": "production",
    "PORT": 3000,
    "ALLOWED_ORIGINS": "https://keykurs.ru,https://www.keykurs.ru"
  },
  "message": "Все необходимые переменные окружения установлены"
}
```

### GET /api/diagnostics
Интерактивная HTML страница с кнопками для всех проверок.

## 🛠️ Распространенные проблемы и решения

### Проблема: "connection refused"
**Решение:** Проверьте DATABASE_URL в файле .env

### Проблема: "relation does not exist"
**Решение:** Примените SQL миграции из папки timeweb-migrations

### Проблема: "password authentication failed"
**Решение:** Проверьте правильность пароля в DATABASE_URL

### Проблема: Таблиц 0
**Решение:** Примените все 4 миграции через SQL редактор Таймвеб
