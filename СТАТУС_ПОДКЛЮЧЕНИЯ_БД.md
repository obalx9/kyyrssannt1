## Причина проблемы

У вас **ДВЕ ПРОБЛЕМЫ**:

### 1. SSL сертификат
Timeweb PostgreSQL использует самоподписанный SSL сертификат, который Node.js `pg` не может верифицировать.

### 2. Неверные учетные данные (наиболее вероятно)
Из предыдущих тестов мы знаем, что при обходе SSL проверки возникает ошибка:
```
password authentication failed for user "gen_user"
Code: 28P01
```

Это означает, что даже если мы решим проблему с SSL, пароль `Global!16` или пользователь `gen_user` всё равно неверны.

---

## Решение

### Шаг 1: Получить корректные учетные данные

**Критически важно!** Вам нужно:

1. Войти в панель Timeweb Cloud
2. Открыть раздел "Базы данных" → PostgreSQL
3. Найти БД с хостом `a6e6285d9957acb308f354f9.twc1.net`
4. **Скопировать или сгенерировать новые:**
   - Имя пользователя
   - Пароль
   - Строку подключения

**БЕЗ КОРРЕКТНЫХ ДАННЫХ ПОДКЛЮЧЕНИЕ НЕ ЗАРАБОТАЕТ!**

### Шаг 2: Обновить DATABASE_URL

После получения данных обновите в файле `.env`:

```env
DATABASE_URL=postgresql://ПРАВИЛЬНЫЙ_USER:ПРАВИЛЬНЫЙ_ПАРОЛЬ@a6e6285d9957acb308f354f9.twc1.net:5432/default_db?sslmode=require
```

**Важно:** Если в пароле есть специальные символы, используйте URL-кодирование:
- `!` → `%21`
- `@` → `%40`
- `#` → `%23`
- `$` → `%24`
- `%` → `%25`
- `&` → `%26`

### Шаг 3: Код API уже обновлен

Я уже обновил `api/index.js` для работы с самоподписанными сертификатами:

```javascript
let sslConfig = false;
if (process.env.NODE_ENV === 'production') {
  sslConfig = { rejectUnauthorized: false };
  console.log('✅ SSL enabled with rejectUnauthorized: false');
}
```

Это решит проблему с SSL после того, как вы укажете правильные учетные данные.

## Архитектура

```
┌─────────────────┐         ┌─────────────────┐         ┌─────────────────┐
│   React App     │────────▶│  Node.js API    │────────▶│  PostgreSQL DB  │
│  (Frontend)     │  HTTP   │   (Backend)     │   pg    │   (Timeweb)     │
│  Port: 5173     │         │   Port: 3000    │         │   Port: 5432    │
└─────────────────┘         └─────────────────┘         └─────────────────┘
     Browser                     Server                      Database
```

### Как это работает:

1. **React → API**: Фронтенд делает HTTP запросы к `http://localhost:3000/api/*`
2. **API → БД**: Express сервер использует `pg` для подключения к PostgreSQL
3. **БД → API → React**: Данные возвращаются через API обратно во фронтенд

### Безопасность:

✅ **Правильно:**
- Креды БД хранятся только в `.env` на сервере API
- React НЕ имеет прямого доступа к БД
- API проверяет JWT токены для аутентификации

❌ **Неправильно (никогда так не делайте):**
- Подключаться к PostgreSQL напрямую из браузера
- Хранить DATABASE_URL в коде React
- Передавать пароли БД в frontend
