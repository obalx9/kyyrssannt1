# Настройка через панель Таймвеб (без терминала)

Эта инструкция для тех, кто не имеет доступа к терминалу и работает только через веб-панель управления Таймвеб.

## Часть 1: Создание и настройка базы данных PostgreSQL

### Шаг 1: Создайте базу данных

1. Войдите в личный кабинет Таймвеб
2. В левом меню выберите **"Базы данных"**
3. Нажмите кнопку **"Создать базу данных"**
4. Выберите тип: **PostgreSQL**
5. Заполните форму:
   - **Имя базы:** kursat_platform (или любое другое)
   - **Версия:** Выберите последнюю доступную (PostgreSQL 14 или 15)
   - **Тарифный план:** Выберите подходящий
6. Нажмите **"Создать"**

### Шаг 2: Получите данные подключения

После создания базы данных:

1. Откройте созданную базу в списке
2. Найдите раздел **"Данные для подключения"**
3. Скопируйте и сохраните:
   - **Хост** (например: `pg-12345.timeweb.cloud`)
   - **Порт** (обычно `5432`)
   - **Имя базы данных**
   - **Имя пользователя**
   - **Пароль**

### Шаг 3: Создайте строку подключения

Используя полученные данные, составьте строку подключения в формате:
```
postgresql://[имя_пользователя]:[пароль]@[хост]:[порт]/[имя_базы]
```

**Пример:**
```
postgresql://user_12345:MyPassword123@pg-12345.timeweb.cloud:5432/kursat_platform
```

Сохраните эту строку - она понадобится для настройки приложения.

## Часть 2: Применение SQL миграций

### Шаг 4: Откройте SQL редактор

1. В панели Таймвеб откройте вашу базу данных
2. Найдите вкладку **"SQL редактор"** или **"Выполнить SQL"**
3. Перед вами откроется текстовое поле для ввода SQL запросов

### Шаг 5: Примените миграцию 1 - Система аутентификации

1. Откройте файл проекта: `timeweb-migrations/01_create_auth_system.sql`
2. Выделите **ВСЁ** содержимое файла (Ctrl+A)
3. Скопируйте (Ctrl+C)
4. Вставьте в SQL редактор Таймвеб (Ctrl+V)
5. Нажмите кнопку **"Выполнить"** или **"Execute"**
6. Дождитесь завершения (должно появиться сообщение об успехе)

**Что создается:**
- Таблица users (пользователи)
- Таблица user_roles (роли пользователей)

### Шаг 6: Примените миграцию 2 - Схема платформы

1. Откройте файл: `timeweb-migrations/02_create_platform_schema.sql`
2. Выделите всё содержимое
3. Скопируйте
4. Вставьте в SQL редактор (можете очистить предыдущий запрос)
5. Нажмите **"Выполнить"**
6. Дождитесь завершения

**Что создается:**
- Таблица sellers (продавцы)
- Таблица students (студенты)
- Таблица courses (курсы)
- Таблица course_posts (посты курсов)
- Таблица course_post_media (медиафайлы)
- Таблица course_enrollments (записи на курсы)
- И другие таблицы

### Шаг 7: Примените миграцию 3 - Политики безопасности

1. Откройте файл: `timeweb-migrations/03_setup_rls_policies.sql`
2. Скопируйте всё содержимое
3. Вставьте в SQL редактор
4. Нажмите **"Выполнить"**

**Что создается:**
- Политики безопасности Row Level Security (RLS)
- Права доступа к данным

### Шаг 8: Примените миграцию 4 - Дополнительные функции

1. Откройте файл: `timeweb-migrations/04_add_additional_features.sql`
2. Скопируйте всё содержимое
3. Вставьте в SQL редактор
4. Нажмите **"Выполнить"**

**Что создается:**
- Дополнительные функции и триггеры
- Оптимизации для работы с данными

### Шаг 9: Проверьте созданные таблицы

В SQL редакторе выполните:
```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

Вы должны увидеть список таблиц:
- course_enrollments
- course_post_media
- course_posts
- courses
- sellers
- student_pinned_posts
- students
- telegram_bots
- telegram_media_group_buffer
- user_roles
- users

**Всего должно быть 11 таблиц.**

## Часть 3: Настройка приложения на хостинге

### Шаг 10: Загрузите файлы на сервер

1. В панели Таймвеб откройте раздел **"Сайты"** или **"Хостинг"**
2. Выберите ваш сайт
3. Откройте **"Файловый менеджер"**
4. Загрузите все файлы проекта в корневую директорию сайта

### Шаг 11: Создайте файл .env

1. В файловом менеджере нажмите **"Создать файл"**
2. Назовите файл `.env` (с точкой в начале!)
3. Откройте файл для редактирования
4. Вставьте следующее содержимое:

```env
DATABASE_URL=postgresql://[ваши_данные_из_шага_3]
JWT_SECRET=ваш_случайный_секрет_минимум_32_символа
NODE_ENV=production
PORT=3000
ALLOWED_ORIGINS=https://ваш-домен.ru
```

**Важно:**
- Замените `DATABASE_URL` на строку подключения из Шага 3
- Для `JWT_SECRET` создайте случайную строку минимум 32 символа (можно использовать генератор паролей)
- Замените `ваш-домен.ru` на ваш реальный домен

**Пример заполненного .env:**
```env
DATABASE_URL=postgresql://user_12345:MyPassword123@pg-12345.timeweb.cloud:5432/kursat_platform
JWT_SECRET=qW8eR7tY6uI5oP4aS3dF2gH1jK9lM0nB3vC4xZ5
NODE_ENV=production
PORT=3000
ALLOWED_ORIGINS=https://kursat-platform.ru
```

5. Сохраните файл

### Шаг 12: Настройте Telegram бота

1. В Telegram найдите @BotFather
2. Создайте нового бота командой `/newbot`
3. Следуйте инструкциям и получите **токен бота**
4. Скопируйте токен (выглядит как `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)
5. Вернитесь в SQL редактор Таймвеб
6. Выполните SQL запрос:

```sql
INSERT INTO telegram_bots (bot_username, bot_token, is_active)
VALUES ('YourBotUsername', 'ваш_токен_от_botfather', true);
```

**Пример:**
```sql
INSERT INTO telegram_bots (bot_username, bot_token, is_active)
VALUES ('KursatPlatformBot', '1234567890:ABCdefGHIjklMNOpqrsTUVwxyz', true);
```

### Шаг 13: Установите зависимости и запустите приложение

Если у вас есть доступ к Node.js на хостинге:

1. Откройте терминал в панели (если доступен)
2. Перейдите в директорию проекта
3. Выполните:
```bash
npm install
npm run build
npm start
```

Если терминал недоступен, используйте раздел "Приложения" в Таймвеб:
1. Создайте новое приложение Node.js
2. Укажите путь к вашему проекту
3. Команда запуска: `npm start`
4. Запустите приложение

## Часть 4: Проверка работоспособности

### Шаг 14: Откройте страницу диагностики

В браузере перейдите по адресу:
```
https://ваш-домен.ru/api/diagnostics
```

На странице будут доступны кнопки для проверки:

1. **Проверить подключение** - должна показать ✅ УСПЕШНО ПОДКЛЮЧЕНО
2. **Показать таблицы** - должна показать 11 таблиц
3. **Проверить конфигурацию** - все переменные должны быть установлены
4. **Запустить полную проверку** - итоговый отчет

### Шаг 15: Проверьте API вручную

Откройте в браузере эти ссылки по очереди:

#### 1. Проверка подключения к БД:
```
https://ваш-домен.ru/api/db-check
```

Должны увидеть:
```json
{
  "status": "connected",
  "database": "kursat_platform",
  "tablesCount": 11
}
```

#### 2. Список таблиц:
```
https://ваш-домен.ru/api/db-tables
```

#### 3. Переменные окружения:
```
https://ваш-домен.ru/api/env-check
```

#### 4. Проверка бота:
```
https://ваш-домен.ru/api/telegram-bot
```

Должен вернуть имя вашего бота.

### Шаг 16: Откройте главную страницу

Перейдите на:
```
https://ваш-домен.ru
```

Должна открыться главная страница платформы с кнопкой "Войти через Telegram".

## Что делать если что-то не работает?

### Проблема: База данных не подключается

**Проверьте:**
1. Правильность строки DATABASE_URL в файле .env
2. Что база данных запущена в панели Таймвеб
3. Что в строке подключения нет лишних пробелов
4. Что пароль указан правильно (без специальных символов в URL нужно использовать URL-кодирование)

### Проблема: Таблиц нет или меньше 11

**Решение:**
1. Вернитесь к Шагам 5-8
2. Убедитесь, что каждая миграция выполнилась успешно
3. Проверьте, не было ли ошибок в SQL редакторе
4. При необходимости выполните миграции заново

### Проблема: Страница /api/diagnostics не открывается

**Возможные причины:**
1. Приложение не запущено - проверьте статус в разделе "Приложения"
2. Неверный домен - проверьте настройки домена в панели
3. Порт занят - измените PORT в .env на другой

### Проблема: JWT_SECRET не установлен

**Решение:**
1. Откройте файл .env через файловый менеджер
2. Добавьте строку: `JWT_SECRET=случайная_строка_32_символа`
3. Сохраните файл
4. Перезапустите приложение

## Полезные SQL запросы для панели

### Проверить количество таблиц:
```sql
SELECT COUNT(*) as table_count
FROM information_schema.tables
WHERE table_schema = 'public';
```

### Посмотреть список таблиц с количеством записей:
```sql
SELECT
  'users' as table_name, COUNT(*) as count FROM users
UNION ALL SELECT 'courses', COUNT(*) FROM courses
UNION ALL SELECT 'sellers', COUNT(*) FROM sellers
UNION ALL SELECT 'students', COUNT(*) FROM students
UNION ALL SELECT 'telegram_bots', COUNT(*) FROM telegram_bots;
```

### Проверить, есть ли боты:
```sql
SELECT bot_username, is_active, created_at
FROM telegram_bots;
```

### Создать тестового пользователя (для отладки):
```sql
INSERT INTO users (telegram_id, telegram_username, first_name)
VALUES (123456789, 'testuser', 'Тест')
RETURNING *;
```
