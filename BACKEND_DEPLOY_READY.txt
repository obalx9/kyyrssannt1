⚡ БЫСТРЫЙ СТАРТ (5 МИНУТ)

✨ ГОТОВЫЕ ЗНАЧЕНИЯ
════════════════════════════════════════════════════════════════════════════════

Окружение:           Node.js 24
Фреймворк:           Express
Команда сборки:      npm install
Команда запуска:     node index.js
Путь проверки:       /health
Ветка:               main

Переменные (5 шт):
1. DATABASE_URL      ← Из Timeweb БД PostgreSQL
2. JWT_SECRET        ← Сгенерируйте (openssl rand -hex 32)
3. NODE_ENV          = production
4. PORT              = 3000
5. ALLOWED_ORIGINS   ← Домены вашего фронтенда

════════════════════════════════════════════════════════════════════════════════
🔑 ПОЛУЧЕНИЕ ПЕРЕМЕННЫХ
════════════════════════════════════════════════════════════════════════════════

DATABASE_URL:
├─ Идите в Timeweb → Облачные БД → PostgreSQL
├─ Откройте вашу базу данных
├─ Скопируйте "Connection String"
└─ Вставьте в переменную

JWT_SECRET:
├─ Linux/Mac:   openssl rand -hex 32
├─ Windows:     [System.Convert]::ToBase64String((1..32|%{[byte](Get-Random -Max 256)}))
├─ Онлайн:      https://www.random.org/strings/
└─ Результат вставьте в переменную

NODE_ENV:
└─ Просто напишите:  production

PORT:
└─ Просто напишите:  3000

ALLOWED_ORIGINS:
├─ Один домен:    https://kursat.com
├─ Много доменов: https://kursat.com,https://app.kursat.com
└─ Примеры в документации

✅ ПРОВЕРКА ПОСЛЕ ДЕПЛОЯ
════════════════════════════════════════════════════════════════════════════════

Когда Timeweb скажет что приложение готово, проверьте:

1. Здоровье:
   https://your-api.timeweb.app/health
   → Должно быть: {"status":"ok","timestamp":"..."}

2. Переменные:
   https://your-api.timeweb.app/api/env-check
   → Должны быть все 5 переменных

3. Подключение к БД:
   https://your-api.timeweb.app/api/db-check
   → Должно быть: "status":"connected"

4. Красивая диагностика (откройте в браузере):
   https://your-api.timeweb.app/api/diagnostics
   → Зелёные галочки везде

🚀 ЧТО ДАЛЬШЕ
════════════════════════════════════════════════════════════════════════════════

После успешного деплоя:

✅ Backend API работает на: https://your-api.timeweb.app
✅ Доступны все эндпоинты: /api/courses, /api/users и т.д.
✅ БД подключена и работает
✅ Telegram авторизация готова
✅ CORS настроен для вашего фронтенда

Если у вас есть отдельный фронтенд приложение:
- Добавьте в его .env: VITE_API_URL=https://your-api.timeweb.app
- Все запросы от фронта пойдут на этот URL
- CORS позволит запрашивать данные (если ALLOWED_ORIGINS правильно настроен)

════════════════════════════════════════════════════════════════════════════════
⚠️ ВАЖНО ПОМНИТЬ
════════════════════════════════════════════════════════════════════════════════

1. DATABASE_URL содержит пароль - это нормально, переменные Timeweb зашифрованы
2. JWT_SECRET должен быть криптостойким - не используйте простые пароли
3. Не коммитьте реальные переменные в .env файлы
4. ALLOWED_ORIGINS должен содержать ТОЧНЫЙ домен вашего фронта (с https://)
5. При обновлении кода в main ветке Timeweb автоматически переделает
