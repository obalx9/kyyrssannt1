## 📂 Структура файлов для деплоя

```
✅ Что должно быть в репозитории:

project/
├── api/
│   └── index.js              ✓ Backend с SSL поддержкой
├── src/                      ✓ Frontend код
├── certs/
│   ├── root.crt             ✓ SSL сертификат (публичный)
│   └── README.md            ✓ Описание
├── timeweb-migrations/      ✓ SQL миграции
├── package.json             ✓ Зависимости
├── vite.config.ts           ✓ Конфигурация сборки
└── .env.example             ✓ Пример переменных

❌ Что НЕ должно быть в репозитории:

├── .env                     ❌ Реальные пароли
├── node_modules/            ❌ Зависимости (устанавливаются автоматически)
└── build/                   ❌ Собранные файлы (генерируются автоматически)
```

## 🆘 Быстрая диагностика

### Если сайт не открывается (502/504):

```
1. Проверьте логи в App Platform
   → Есть ли ошибки?

2. Проверьте команду запуска
   → Должна быть: npm start

3. Проверьте PORT
   → Должен быть: 3000
```

### Если БД не подключается:

```
1. Откройте /api/diagnostics
   → Нажмите "Проверить подключение"

2. Проверьте DATABASE_URL
   → Правильный ли пароль?
   → Нет ли ?sslmode=verify-full?

3. Проверьте логи
   → Загружен ли SSL сертификат?
```

### Если миграции не работают:

```
1. Проверьте /api/db-check
   → БД подключена?

2. Примените миграции по порядку
   → 01, 02, 03, 04

3. Проверьте /api/db-tables
   → Есть ли таблицы?
```
