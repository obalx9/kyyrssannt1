# 🤔 FAQ: Backend API на Timeweb

## ❓ Общие вопросы

### Q: Какую версию Node.js выбрать?
**A:** 24 (LTS) - это стабильная версия, рекомендуется для продакшена.

### Q: Какой фреймворк? Express?
**A:** Да, Express уже используется в проекте. Не меняйте.

### Q: Нужно ли отдельно деплоить backend от frontend?
**A:** Да! Они деплоятся как два отдельных приложения на Timeweb Cloud Apps:
- Backend API на одном приложении
- Frontend (React) на другом приложении
- Они общаются через HTTP API

### Q: Можно ли деплоить обе части вместе?
**A:** Технически можно, но не рекомендуется. Разделение позволяет:
- Независимо обновлять фронт и бэк
- Масштабировать отдельно
- Использовать разные домены/сертификаты

## 🔑 Переменные окружения

### Q: Как получить DATABASE_URL?
**A:**
1. Зайдите в Timeweb → Облачные БД
2. Выберите вашу БД PostgreSQL
3. Нажмите на неё
4. Найдите "Connection String" или "Строка подключения"
5. Скопируйте полностью (включая пароль)
6. Вставьте в переменную DATABASE_URL

### Q: DATABASE_URL содержит пароль - это безопасно?
**A:** Да, это безопасно в Timeweb, потому что:
- Переменные хранятся зашифрованными
- Доступны только внутри контейнера приложения
- Не попадают в логи
- Не видны другим приложениям

### Q: Где взять JWT_SECRET?
**A:** Генерируйте сами! Это может быть любая криптостойкая строка:

```bash
# Linux/Mac:
openssl rand -hex 32

# Windows PowerShell:
[System.Convert]::ToBase64String((1..32 | ForEach-Object { [byte](Get-Random -Maximum 256) }))

# Онлайн:
https://www.random.org/strings/ → длина 64, букв и цифр
```

### Q: Что будет если JWT_SECRET угадают?
**A:** Они смогут подделывать токены и получить доступ. Поэтому:
- Генерируйте сложный, случайный JWT_SECRET
- Никогда не пишите его в коде
- Меняйте на регулярной основе
- Храните только в переменных Timeweb

### Q: ALLOWED_ORIGINS - это что?
**A:** CORS защита. Указывает какие домены могут делать запросы к API.

Примеры:
```
# Один домен:
https://kursat.com

# Несколько:
https://kursat.com,https://app.kursat.com

# С тестированием (небезопасно!):
https://kursat.com,http://localhost:5173

# Все (не рекомендуется!):
*
```

### Q: У меня есть несколько фронтенд доменов?
**A:** Добавьте их все в ALLOWED_ORIGINS, разделённые запятой без пробелов:

```
https://main.kursat.com,https://app.kursat.com,https://admin.kursat.com
```

## 🚀 Деплой

### Q: Сколько времени занимает деплой?
**A:** Обычно 2-5 минут:
- 1 мин - сборка окружения
- 1-2 мин - установка зависимостей (npm install)
- 0.5-1 мин - запуск приложения

### Q: Приложение помещается в контейнер, правда?
**A:** Да, Timeweb использует Docker контейнеры. Команда запуска запускается внутри контейнера.

### Q: Что означает `pm2 start --no-daemon`?
**A:**
- `pm2` - менеджер процессов Node.js
- `--no-daemon` - запускает в foreground (не фоновый процесс)
- Это нужно для контейнеров, чтобы контейнер не умирал
- Важно: не убирайте `--no-daemon`!

### Q: Что будет если приложение упадёт?
**A:** PM2 его перезагрузит автоматически.

### Q: Как обновить приложение?
**A:**
1. Делаете commit в GitHub (ветка main)
2. Пушите: `git push origin main`
3. Timeweb видит обновление → сам переделовывает
4. Готово! (максимум 5 минут)

Не нужно вручную ничего делать!

## 🔗 API и интеграция

### Q: Какой URL будет у моего API?
**A:**
```
https://[project-name].timeweb.app
```

Точный URL будет в панели Timeweb после деплоя.

### Q: Как проверить что API работает?
**A:**
```bash
curl https://your-api.timeweb.app/health

# Должно вернуть:
{"status":"ok","timestamp":"2025-02-18T..."}
```

### Q: Как подключить фронтенд к API?
**A:** В фронтенде (.env):

```
VITE_API_URL=https://backend-api.timeweb.app
```

А в коде:
```javascript
const response = await fetch(
  `${import.meta.env.VITE_API_URL}/api/courses`,
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);
```

### Q: Как авторизоваться в API?
**A:**
1. Пользователь входит через Telegram
2. Фронтенд отправляет: `POST /api/telegram-auth` с данными Telegram
3. Backend возвращает JWT токен
4. Фронтенд сохраняет токен
5. Для остальных запросов: `Authorization: Bearer JWT_TOKEN`

### Q: Где хранить JWT токен на фронтенде?
**A:**
- **Лучше всего:** localStorage (простенько и сойдёт)
- **Более безопасно:** sessionStorage (очищается при закрытии браузера)
- **Максимально безопасно:** в cookies с httpOnly флагом (требует server-side рендеринга)

## 🐛 Отладка и проблемы

### Q: Приложение не стартует, что делать?
**A:**
1. Откройте логи: Timeweb → Приложение → Логи
2. Ищите красные ошибки
3. Частые причины:
   - DATABASE_URL неправильный
   - npm install завалился (зависимостью)
   - Синтаксическая ошибка в коде

### Q: Как посмотреть логи?
**A:**
```
Timeweb → Облачные приложения → Ваше приложение → Логи
```
Там вся история - ошибки, предупреждения, info.

### Q: 502 Bad Gateway - что это?
**A:** Приложение не отвечает или ещё не стартовало.

Решение:
- Подождите 1-2 минуты (деплой может быть в процессе)
- Проверьте логи на ошибки
- Проверьте переменные окружения
- Перезагрузите приложение

### Q: Cannot connect to database
**A:**
1. Проверьте DATABASE_URL правильный (скопируйте заново из Timeweb БД)
2. Проверьте БД доступна (не отключена, не заблокирована)
3. Проверьте пароль верный
4. Если на локальной машине работает, значит в Timeweb переменная неправильно передана

### Q: CORS error - Origin not allowed
**A:**
1. Проверьте ALLOWED_ORIGINS (должно быть ваше доменное имя фронта)
2. Проверьте URL фронта точно совпадает (с https://, без слешей)
3. Если несколько доменов - разделите запятой без пробелов

### Q: Как включить debug режим?
**A:** Добавьте переменную:

```
DEBUG=express:*
```

Тогда в логах будет больше информации.

### Q: Приложение постоянно перезагружается
**A:**
1. Проверьте логи - какая ошибка?
2. Обычно - ошибка в коде или неправильная переменная окружения
3. Исправьте ошибку → пушьте в main → переделовывает

## 📊 Мониторинг

### Q: Как следить за здоровьем приложения?
**A:** Проверяйте эти эндпоинты:

```bash
# Здоровье (простая проверка)
curl https://your-api.timeweb.app/health

# Конфигурация (какие переменные установлены)
curl https://your-api.timeweb.app/api/env-check

# Статус БД (подключена ли база)
curl https://your-api.timeweb.app/api/db-check

# Структура БД (сколько таблиц, сколько записей)
curl https://your-api.timeweb.app/api/db-tables

# Красивая диагностика (откройте в браузере)
https://your-api.timeweb.app/api/diagnostics
```

### Q: Как узнать сколько памяти使用旅application?
**A:** В панели Timeweb в разделе "Мониторинг" (если есть на вашем тарифе).

### Q: Как узнать сколько запросов в минуту?
**A:** Официально не может узнать, но можно:
- Логировать запросы вручную в коде
- Использовать сторонние сервисы мониторинга (Sentry, New Relic)
- Смотреть трафик в панели Timeweb

## 🔄 Обновления

### Q: Как обновить версию Node.js?
**A:**
1. В панели Timeweb → Приложение → Настройки
2. Измените версию
3. Приложение переделается с новой версией

### Q: Как обновить зависимости?
**A:**
1. Локально: `npm update` → `git commit` → `git push`
2. Timeweb автоматически переделает

### Q: Нужно ли менять package-lock.json?
**A:** Нет, коммитьте как есть. npm install использует его для консистентности.
